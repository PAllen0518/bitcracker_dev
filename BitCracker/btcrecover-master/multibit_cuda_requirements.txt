multibit_cuda_threads.exe — Build Requirements
===============================================

CUDA Toolkit
  Version : 13.0 (built Jul 16 2025)
  Path    : C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.0
  Download: https://developer.nvidia.com/cuda-downloads

GPU
  Tested on : NVIDIA GeForce RTX 2060 (compute capability sm_75)
  Driver    : 580.97 or later (must support CUDA 13.0)
  Throughput: ~11M passwords/sec on search46.txt

Host Compiler
  Visual Studio 2022 Build Tools v17.14 (MSVC 14.44)
  IMPORTANT: VS 2026 (MSVC 19.51+) is NOT supported by CUDA 13.0
  Download: https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio-2022
  Component required: "Desktop development with C++"

Windows SDK
  Version : 10.0.22621.0 (Windows 11 22H2)
  Install via VS installer > Individual components > "Windows 11 SDK (10.0.22621.0)"
  NOTE: SDK 10.0.26100.0 causes "Host compiler targets unsupported OS" error

Known Build Issues
  1. Username with spaces (e.g. "Paul Allen") causes nvcc to generate temp file
     paths without quotes, which breaks cl.exe.
     Fix: set TEMP=C:\Temp and TMP=C:\Temp before compiling (done in build_cuda.bat,
     override with the BUILD_TEMP_DIR env var if C:\Temp isn't suitable).

  1a. build_cuda.bat's toolchain paths are overridable via environment
      variables if your CUDA/VS install locations differ from the defaults
      below: CUDA_TOOLKIT_DIR (CUDA Toolkit root), VS_TOOLS_BINDIR (MSVC
      host-compiler bin dir), BUILD_TEMP_DIR (nvcc temp dir).

  2. CUDA 13.0 does not support VS 2026 (MSVC 19.51). Must use VS 2022.

  3. If multibit_cuda_threads.exe is running, the linker cannot overwrite it.
     Stop the process before rebuilding.

  4. If PATH contains CUDA v12.9 before v13.0, the wrong nvcc may be used.
     build_cuda.bat uses the full path to nvcc.exe to avoid this.

  5. VS 2022 Build Tools + CUDA 13.0 requires --allow-unsupported-compiler
     because the host_config.h version check in CUDA 13.0 is conservative.
     This is safe: MSVC 14.44 is fully compatible in practice.

Build Command
  Run build_cuda.bat from the btcrecover-master directory.
  This builds both multibit_cuda.exe and multibit_cuda_threads.exe.

  Key flags used:
    -arch=sm_75                    RTX 2060 compute capability
    --allow-unsupported-compiler   bypass conservative MSVC version check
    --compiler-bindir <VS2022 path>  use VS 2022 cl.exe, not VS 2026
    -std=c++14                     required for std::thread, lambdas
    -O3                            full optimisation
